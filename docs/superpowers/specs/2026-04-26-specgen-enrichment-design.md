# Specgen Enrichment — Per-Op Metadata + Spec-Level Info + Real-Type Test

**Status:** Design approved · Implementation pending
**Date:** 2026-04-26
**Issues:** [#60](https://github.com/kaio6fellipe/event-driven-bookinfo/issues/60), [#61](https://github.com/kaio6fellipe/event-driven-bookinfo/issues/61)
**Scope:** `tools/specgen/`, `pkg/api`, `pkg/events`, all 6 service spec artifacts

## Problem

Two related quality gaps in the API spec generation pipeline shipped in PRs #55 / #58 / #59:

**[#60] Spectral warnings — 46 across the 6 generated specs.** Categories:

- **33 per-operation** (11 OpenAPI operations × `tags`, `operationId`, `description`)
- **6 OpenAPI top-level**: `servers`, `info.description`, `info.contact`
- **6 AsyncAPI top-level**: `servers`, `info.description`, `info.contact`, `info.license`, `tags`, per-operation description
- **1 informational** (AsyncAPI 3.0.0 → 3.1.0 latest-version note)

These don't fail CI today but degrade Backstage / Swagger UI documentation quality and would be a problem if a future linter ruleset upgrades them to errors.

**[#61] Walker fixture redeclares types locally.** `tools/specgen/testdata/fixture/api/endpoints.go` and `events/exposed.go` declare their own `Endpoint`/`Descriptor` rather than importing `pkg/api`/`pkg/events`. The walker matches by AST field name, so the fixture works coincidentally — but a field rename in the real types (e.g. `EventName` → `CommandName`) would silently pass fixture tests while breaking real-service generation.

## Goals

- Reduce `make lint-specs` warning count from 46 to ≤1 (just the structural ones we can't fix from metadata).
- Add `OperationID`, `Tags`, `Description` fields to `pkg/api.Endpoint`; `Tags` to `pkg/events.Descriptor`. All optional with smart-default fallbacks in the generator.
- Hardcode org-level metadata (license, contact, servers) in the runner — avoids per-service duplication.
- Bump `asyncapi: 3.0.0` → `3.1.0`.
- Add a walker integration test against a real service that catches field renames.
- Per-service code change footprint: zero. The hybrid defaults close all warnings without any service-side endpoints.go/exposed.go change.

## Non-goals

- Per-service service descriptions written by humans. The default ("`<serviceName>` — generated by tools/specgen") is acceptable for v1; teams can override with explicit `Description` constants in a follow-up if desired.
- Backstage TechDocs, Swagger UI hosting, or any consumer-side rendering work. This PR closes warnings only.
- Removing the synthetic walker fixture entirely. Unit tests still want a tiny self-contained fixture for fast edge-case coverage.
- Pact / contract testing harness. (Pre-existing deferred non-goal from PR #55.)

## Smart-default logic (in the generator builders, not in `pkg/api`)

The fallback logic lives in `tools/specgen/internal/openapi/openapi.go` and `tools/specgen/internal/asyncapi/asyncapi.go`. Pseudocode:

```go
operationID := ep.OperationID
if operationID == "" {
    operationID = lowerCamelCase(ep.Method, ep.Path)
    // e.g. ("GET", "/v1/ratings/{id}") → "getRatingsById"
    // e.g. ("POST", "/v1/ratings")     → "submitRatings" (POST + plural noun → "submit")
    //                                  Actual rule: lowerCamelCase(method) + cleaned-path-segments
    //                                  Concrete: ("POST", "/v1/ratings") → "postV1Ratings"
}

description := ep.Description
if description == "" {
    description = ep.Summary
}

tags := ep.Tags
if len(tags) == 0 {
    tags = []string{in.ServiceName}
}
```

`lowerCamelCase` is a small helper that:
1. Strips `{}` from path segments.
2. Splits the path on `/` and filters empties.
3. Joins method + path segments and lowerCamelCases.

So `("GET", "/v1/ratings/{id}")` → `getV1RatingsId` (predictable, unique per route).

## Walker integration test (`#61` resolution)

New file: `tools/specgen/internal/walker/integration_test.go`

```go
package walker_test

import (
    "os"
    "path/filepath"
    "testing"

    "github.com/kaio6fellipe/event-driven-bookinfo/tools/specgen/internal/walker"
)

const realModulePath = "github.com/kaio6fellipe/event-driven-bookinfo"

func realRepoRoot(t *testing.T) string {
    t.Helper()
    p, err := filepath.Abs("../../../..")
    if err != nil { t.Fatal(err) }
    if _, err := os.Stat(filepath.Join(p, "go.mod")); err != nil {
        t.Fatalf("not in expected repo layout: %v", err)
    }
    return p
}

// TestLoadEndpoints_RealRatingsService catches field renames in pkg/api.Endpoint
// that the walker silently ignores. The synthetic fixture under testdata/ uses
// locally-redeclared types and would not catch a rename.
func TestLoadEndpoints_RealRatingsService(t *testing.T) {
    repoRoot := realRepoRoot(t)

    endpoints, version, err := walker.LoadEndpoints(repoRoot,
        realModulePath+"/services/ratings/internal/adapter/inbound/http")
    if err != nil {
        t.Fatalf("LoadEndpoints: %v", err)
    }
    if version == "" {
        t.Error("expected APIVersion to be a non-empty string")
    }
    if len(endpoints) < 2 {
        t.Fatalf("expected ≥2 endpoints, got %d", len(endpoints))
    }

    // Find the POST /v1/ratings endpoint and assert on its known fields.
    var post walker.EndpointInfo
    for _, ep := range endpoints {
        if ep.Method == "POST" && ep.Path == "/v1/ratings" {
            post = ep
            break
        }
    }
    if post.Method != "POST" {
        t.Fatal("POST /v1/ratings not found in endpoints")
    }
    if post.EventName != "rating-submitted" {
        t.Errorf("EventName = %q, want rating-submitted", post.EventName)
    }
    if post.RequestType == nil {
        t.Error("RequestType expected non-nil")
    }
    if post.ResponseType == nil {
        t.Error("ResponseType expected non-nil")
    }
}

// TestLoadExposed_RealRatingsService catches field renames in pkg/events.Descriptor.
func TestLoadExposed_RealRatingsService(t *testing.T) {
    repoRoot := realRepoRoot(t)

    exposed, err := walker.LoadExposed(repoRoot,
        realModulePath+"/services/ratings/internal/adapter/outbound/kafka")
    if err != nil {
        t.Fatalf("LoadExposed: %v", err)
    }
    if len(exposed) < 1 {
        t.Fatal("expected ≥1 exposed descriptor")
    }

    d := exposed[0]
    if d.Name != "rating-submitted" {
        t.Errorf("Name = %q, want rating-submitted", d.Name)
    }
    if d.CEType != "com.bookinfo.ratings.rating-submitted" {
        t.Errorf("CEType = %q, want com.bookinfo.ratings.rating-submitted", d.CEType)
    }
    if d.PayloadType == nil {
        t.Error("PayloadType expected non-nil")
    }
}
```

These tests are gated by the existence of `services/ratings/...` so they `t.Skip` if run in a context where the service is absent (e.g. moving the walker to a separate repo someday).

## Per-service migration footprint

**Zero**. None of the 6 services' `endpoints.go` or `exposed.go` files need changes. The hybrid defaults close every warning automatically:

- Each service's operations get auto-tagged `[serviceName]`, auto-described from `Summary`, auto-id'd from method+path.
- The runner-level `Metadata` is applied to every spec without per-service input.

A future PR can sprinkle explicit `Tags`, `OperationID`, or richer `Description` values on individual endpoints if a service wants to override (e.g. tag a `GET /v1/healthz` with `["operations"]` instead of `["ratings"]`). Out of scope here.

## Verification plan

1. `make lint-specs` — should drop from **46 warnings** to **0 warnings** (the asyncapi-latest-version note disappears with the 3.1 bump; per-op metadata closes the 33; spec-level metadata closes the 12). Acceptable to leave 1-2 if they're structural (e.g. unused-component) and not metadata-related.
2. `make generate-specs && git diff --exit-code` — drift gate clean after regen.
3. `make diff-specs` — will flag breaking changes (added `info.contact`, `info.license`, `servers`, per-op `tags`/`operationId`/`description`, AsyncAPI 3.1.0 version bump). The `breaking-change` PR label triggers advisory mode; expected.
4. `golangci-lint run ./...` — 0 issues.
5. `go test ./... -race -count=1` — all 321+ tests pass; new walker integration tests included.
6. `make stop-k8s && make run-k8s` clean cycle + smoke (per the standing rule from PR #63 brainstorm). No runtime regression — the runtime contract is unchanged; only generated docs differ.

## Risks & open questions

- **AsyncAPI 3.1.0 backward compat**: 3.1 is fully backward-compatible with 3.0 channels/messages. The existing parsers and Backstage AsyncAPI plugin both support 3.1. Risk: low; verified by spectral lint passing on the regenerated specs.
- **Spectral may add new warnings on 3.1.0**: when the generated AsyncAPI version bumps, spectral may surface rules that didn't apply to 3.0. We'll address in follow-up if any new errors appear; warnings remain acceptable per PR #55's policy.
- **`postV1Ratings` style operationIds** are functional but not pretty. Future PR can map common patterns (e.g. POST + collection → "create" + singular) — out of scope.
- **Server URLs are local-dev-only** (`http://localhost:8080`, k3d kafka). Production URLs would need per-environment overrides. Out of scope; the spec is documentation, not deployment config.
- **Walker integration test couples to ratings**: if ratings's endpoints.go is refactored, the integration test breaks. This is the desired property (catches changes that affect the walker), but call-out for future maintainers.
